# Text Input

A standard method of collecting user-inputted information.

Text inputs can be rendered without a label, but this should only be done if the
form requirements are clear.

```jsx react-live
<TextInput label="Text Input" placeholder="e.g. username" />
```

## Types

Text inputs support a range of input formats, some of which include automatic
browser validation. These can be set by providing HTML input `type` attributtes:

```jsx react-live
<Form maxWidth="25rem">
  <FormItem>
    <TextInput type="text" label="Text" placeholder="e.g. username" required />
  </FormItem>
  <FormItem>
    <TextInput type="date" label="Date" />
  </FormItem>
  <FormItem>
    <TextInput
      type="email"
      label="Email"
      placeholder="e.g. jake.weary@sportsbet.io"
      required
    />
  </FormItem>
  <FormItem>
    <TextInput type="number" label="Number" placeholder="e.g. 1234" required />
  </FormItem>
  <FormItem>
    <TextInput
      type="password"
      placeholder="Please enter your account password"
      label="Password"
    />
  </FormItem>
  <FormItem>
    <TextInput type="search" label="Search" placeholder="e.g. EPL" required />
  </FormItem>
  <FormItem>
    <TextInput
      type="tel"
      label="Telephone"
      placeholder="e.g. +372 123 4567"
      required
    />
  </FormItem>
  <FormItem>
    <TextInput
      type="url"
      label="URL"
      placeholder="e.g. https://sportsbet.io"
      required
    />
  </FormItem>
</Form>
```

## States

Text inputs offer different states to provide instant feedback to the user.

### Validation

Errors are automatically handled as part of the HTML input type validation, and
can also be manually triggered via the `error` prop.

```jsx react-live
<Form maxWidth="25rem">
  <FormItem>
    <TextInput
      type="email"
      label="Email (with default validation)"
      placeholder="e.g. jake.weary@sportsbet.io"
      required
    />
  </FormItem>
  <FormItem>
    <TextInput
      type="email"
      label="Email (with error prop)"
      placeholder="e.g. jake.weary@sportsbet.io"
      error
    />
    <FormError>Please enter a valid email address.</FormError>
  </FormItem>
</Form>
```

### Disabled

Passing a standard HTML `disabled` attribute will render an input disabled,
preventing user interaction.

```jsx react-live
<Form maxWidth="25rem">
  <FormItem>
    <TextInput
      type="text"
      label="Disabled Input"
      placeholder="e.g. username"
      disabled
    />
  </FormItem>
</Form>
```
